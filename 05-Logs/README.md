# 05-Logs Directory Structure

This directory contains all project logs, notes, and documentation organized by type and date.

## Directory Structure

### `/Daily/`
Daily session notes and implementation logs organized by date. Files follow the pattern `YYYY-MM-DD-description.md`.

**Contents:**
- Cursor session notes (`cursor-notes.md`)
- Implementation verification logs
- Feature-specific implementation notes
- Bug fix documentation
- Verification and testing notes

### `/Development/`
Development update posts for the forum. These are the public-facing changelogs that get posted to the Development section.

**Contents:**
- `2026-01-21-development-post-01.md` - Initial release
- `2026-01-21-development-post-01-devlogform.md` - Devlog form version
- `2026-01-23-development-post-03.md` - Development Update #3
- `2026-01-25-development-post-04.md` - Development Update #4
- Development update changelogs and review notes

### `/Implementation/`
High-level implementation summaries, completion notes, and verification documents.

**Contents:**
- Complete implementation summaries
- Final verification documents
- Implementation review notes
- Implementation completion status

### `/Reference/`
Reference documents for build commands, migration instructions, and other operational guides.

**Contents:**
- Build success and preview commands
- Migration instructions
- Other reference documentation

## File Naming Conventions

- **Daily logs**: `YYYY-MM-DD-description.md` (e.g., `2026-01-25-cursor-notes.md`)
- **Development posts**: `YYYY-MM-DD-development-post-NN.md` (e.g., `2026-01-25-development-post-04.md`)
- **Implementation summaries**: Descriptive names (e.g., `COMPLETE-IMPLEMENTATION-SUMMARY.md`)
- **Reference docs**: UPPERCASE descriptive names (e.g., `MIGRATION-INSTRUCTIONS.md`)

## Quick Navigation

- **Latest daily notes**: Check `/Daily/` sorted by date (most recent first)
- **Latest development update**: Check `/Development/` sorted by date
- **Build/deployment help**: Check `/Reference/BUILD-SUCCESS-AND-PREVIEW-COMMANDS.md`
- **Migration help**: Check `/Reference/MIGRATION-INSTRUCTIONS.md`
- **Implementation status**: Check `/Implementation/` for summaries and reviews
